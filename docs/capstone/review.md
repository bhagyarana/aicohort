# Capstone Review

Present your project and receive feedback.

## Presentation Format

### Structure (10 minutes)

| Section | Time | Content |
|---------|------|---------|
| Introduction | 1 min | What problem does it solve? |
| Demo | 4 min | Live demonstration |
| Architecture | 2 min | Technical design |
| Challenges | 2 min | Problems faced and solutions |
| Q&A | 1 min | Questions |

## Demo Checklist

Before your demo:

- [ ] System is running locally
- [ ] All dependencies installed
- [ ] Test queries prepared
- [ ] Fallback plan if live demo fails

## Evaluation Rubric

### Functionality (30%)

| Score | Criteria |
|-------|----------|
| Excellent | All features work flawlessly |
| Good | Core features work, minor issues |
| Acceptable | Basic features work |
| Needs Work | Significant functionality gaps |

### Architecture (25%)

| Score | Criteria |
|-------|----------|
| Excellent | Clean, scalable, well-organized |
| Good | Logical structure, some improvements possible |
| Acceptable | Works but could be cleaner |
| Needs Work | Disorganized, hard to understand |

### Code Quality (20%)

| Score | Criteria |
|-------|----------|
| Excellent | Well-documented, typed, tested |
| Good | Mostly documented, some tests |
| Acceptable | Basic documentation |
| Needs Work | Undocumented, no tests |

### Innovation (15%)

| Score | Criteria |
|-------|----------|
| Excellent | Creative, unique approaches |
| Good | Some innovative elements |
| Acceptable | Standard implementation |
| Needs Work | Minimal effort |

### Presentation (10%)

| Score | Criteria |
|-------|----------|
| Excellent | Clear, engaging, professional |
| Good | Well-organized, informative |
| Acceptable | Covers the basics |
| Needs Work | Unclear, unprepared |

## Common Questions

Be prepared to answer:

1. **Why did you choose this architecture?**
2. **What was the hardest part?**
3. **How would you scale this?**
4. **What would you do differently?**
5. **How do you handle errors?**

## Feedback Session

After presentations:

### Peer Feedback

- What worked well?
- What could be improved?
- Ideas for extensions?

### Self-Reflection

- What did you learn?
- What surprised you?
- What would you explore next?

## Post-Review

### Iterations

Based on feedback:
1. Document suggested improvements
2. Prioritize changes
3. Implement updates if time permits

### Portfolio Addition

Your capstone is a great portfolio piece:
- Clean up the code
- Add comprehensive README
- Create a demo video
- Deploy if possible
