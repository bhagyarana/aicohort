## Timeline

| Phase | Duration | Activity |
|-------|----------|----------|
| **Requirements** | Day 1 | Understand requirements, plan approach |
| **Development** | Days 2-5 | Build the system |
| **Testing** | Day 6 | Test and refine |
| **Review** | Day 7 | Present and discuss |

## Required Components

Your project must include:

### 1. Knowledge Integration (Module 2)
- Document ingestion
- Vector store setup
- RAG pipeline

### 2. Agent Workflow (Module 3)
- LangGraph implementation
- State management
- Conditional routing

### 3. Tool Integration (Module 4)
- Custom tools
- API integrations
- Error handling

### 4. Multi-Agent Coordination (Pattern 3)
- Multiple specialized agents
- Task delegation
- Result aggregation

## Project Options

Choose one of these project themes or propose your own:

| Theme | Description | Complexity |
|-------|-------------|------------|
| **Research Assistant** | Research topics, summarize findings, answer questions | Medium |
| **Code Assistant** | Analyze code, suggest improvements, generate tests | Medium |
| **Customer Support Bot** | Answer FAQs, escalate issues, track tickets | Medium |
| **Content Creator** | Research, write, edit, publish content | High |
| **Data Analyst** | Query data, analyze trends, generate reports | High |

## Evaluation Criteria

| Criterion | Weight | Description |
|-----------|--------|-------------|
| **Functionality** | 30% | Does it work correctly? |
| **Architecture** | 25% | Well-designed system? |
| **Code Quality** | 20% | Clean, documented code? |
| **Innovation** | 15% | Creative solutions? |
| **Presentation** | 10% | Clear explanation? |

## Getting Started

1. Read the [Requirements](./requirements) carefully
2. Plan your [Implementation](./implementation) approach
3. Build iteratively, testing as you go
4. Prepare for the [Review](./review) session

:::tip Pro Tip
Start simple and iterate. A working basic system is better than an incomplete complex one.
:::
